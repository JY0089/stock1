# 네이버 오픈API 종류 통합본 (로그인 방식 및 비로그인 방식 전체)

네이버의 오픈API 아키텍처는 데이터의 성격이나 사용자의 개인 정보 열람 필요 유무, 정보 작성의 권한 인증 여부에 따라서 크게 '로그인 방식 오픈 API'와 '비로그인 방식 오픈 API' 두 가지 큰 구조로 구분되어 제공됩니다. 각각의 방식은 헤더에 제공해야 할 토큰이나 자격 증명이 본질적으로 상이하기 때문에 프로젝트 구현 전 확실한 이해가 필요합니다. 본 문서는 비로그인 방식 API와 데이터랩/검색 등에 초점을 맞추되 전체 API의 종류를 온전하게 요약 없이 나열하고 설명합니다.

## 로그인 방식 오픈 API 상세 정리
로그인 방식 오픈 API는 기본적으로 '네이버 로그인' 시스템 인증을 사용자로부터 획득하여 발급되는 사용자별 고유 식별 암호인 접근 토큰(Access Token)을 필수적으로 갖추고 있어야만 호출이 가능한 오픈 API의 형태입니다. 데이터를 단순 조회하는 것을 넘어서 사용자 개개인의 프로필 열람, 카페 게시물 작성, 개인 캘린더 등록 등의 활동에서 필수적입니다.
API 호출 시점에 HTTP Authorization 또는 지정된 매개변수를 통해 네이버 로그인 API로 습득한 Access Token 암호값을 전송 및 증빙해야만 합니다.

- **네이버 로그인 (Naver Login)**: 별도의 복잡한 회원가입 절차 없이, 사용자가 기존에 보유하고 있는 네이버 아이디와 비밀번호만으로 간편하고 안전하게 외부 서비스 프로젝트에 인증을 수행하고 로그인할 수 있도록 릴레이 역할을 제공합니다.
- **카페 (Cafe)**: 사용자가 가입한 네이버 카페들의 목록을 조회하거나 외부 서비스 사이트를 벗어나지 않은 상태에서 특정 네이버 카페에 가입을 시도할 수 있습니다. 또한 인게임이나 특정 화면에서 다이렉트로 게시글과 사진을 등록할 수 있게 권한을 열어주는 API입니다.
- **캘린더 (Calendar)**: 외부 애플리케이션에서 예약한 스케쥴이나 약속, 이벤트 일정 자체를 사용자의 실제 계정 내 네이버 캘린더에 연동하고 다이렉트로 등록해주는 등의 동기화 조정을 할 수 있습니다.

*주요 호출 구조 예:*
`https://nid.naver.com/oauth2.0/authorize` (인증창)
`https://nid.naver.com/oauth2.0/token` (토큰 획득)
`https://openapi.naver.com/v1/nid/me` (기본정보)

## 비로그인 방식 오픈 API (데이터랩, 검색 핵심) 상세 정리
비로그인 방식 오픈 API는 별도로 어떠한 사용자로부터 로그인 과정을 거치거나 개별 접근 권한 토큰 획득 메커니즘을 연동할 필요가 없는 방식입니다. 오로지 HTTP 프로토콜을 호출할 때 헤더(Header) 내부에 네이버 개발자 센터에서 발급 받은 `X-Naver-Client-Id` 값과 `X-Naver-Client-Secret` 값 두 가지만을 텍스트로 직접 전송하는 것만으로 거의 실시간에 가까운 서버 허가를 득할 수 있습니다. 즉시 응답 가능한 편리한 수단입니다.

### 1. 데이터랩 (DataLab) API
네이버 데이터랩 포털 자체에서 제공하는 강력한 통계 데이터를 외부 플랫폼으로 쉽게 반입시키는 도구입니다.
데이터랩 서비스 기능 중 특정 검색어에 대한 검색 트렌드 데이터 리소스(날짜, 기간, 성별, 연령 기준)와 특정 카테고리가 쇼핑/이커머스 영역 내에서 얼마나 자주 클릭되고 노출되었는지를 판단하는 쇼핑 인사이트 통계 정보를 다이나믹하게 수집하실 수 있습니다.

*관련 주요 URL 및 명세 형태:*
- `https://openapi.naver.com/v1/datalab/search` (통합 검색어 트렌드 분석치)
- `https://openapi.naver.com/v1/datalab/shopping/categories` (분야별 대표 카테고리 클릭률 분석치)
- 카테고리 기기별(`device`), 성별(`gender`), 연령대(`age`) 조합의 모든 하위 트리 URL 매핑 지원.

### 2. 검색 (Search) API
네이버 통합 페이지에서 사용자가 직접 검색 바에 타이핑한 결과처럼, 뉴스 기사, 온라인 백과사전 콘텐츠, 매일 올라오는 수 만 건의 블로그 글, 지식iN 답변 내용, 책 서적 등의 결과를 분야별로 디테일하게 호출할 수 있는 최고의 리소스 탐색기입니다. 그뿐만 아니라 위치 기반 지역 결과나 오타 교정 기능, 검색어 성인 판별 지원도 연계됩니다.

*분야별 주요 요청 URL 엔드포인트 명세:*
- 뉴스: `https://openapi.naver.com/v1/search/news`
- 백과사전: `https://openapi.naver.com/v1/search/encyc`
- 블로그 전문 검색: `https://openapi.naver.com/v1/search/blog`
- 쇼핑(스마트스토어 등) 상품: `https://openapi.naver.com/v1/search/shop`
- 웹 문서, 이미지, 전문 정보(doc), 지식iN(kin), 책(book), 카페 글(cafearticle) 등 10여 가지 별도 분리된 엔드포인트 세트 제공.

### 3. 유틸리티 / 소셜 API 등 그 외 비로그인 방식 자원
- **이미지 캡차 (Image Captcha)**: 어뷰징 차단을 위한 네이버 서비스의 강력한 캡차 알고리즘 이미지를 서드파티에서도 사용 가능하게 뚫어줍니다. 기능 별로 `ncaptcha.bin` 등으로 생성과 검증 호출 구간이 분리되어 있습니다.
- **음성 캡차 (Sound Captcha)**: 시각 데이터에 접근하기 어려운 환경을 위하여 오디오 음성 피드백을 통해 캡차 결과를 제출하게 돕는 서브 API입니다 (`scaptcha` 엔드포인트 활용).
- **네이버 공유하기**: 서드파티 웹 브라우저나 앱상의 핵심 기사나 이미지 단일 객체를 곧바로 공유하기 버튼 액션을 통해 사용자 단상 네이버 블로그/카페 그리고 단종된 플랫폼이었던 폴라(PHOLAR) 등에 발행 공유할 수 있게 돕습니다. URL은 `http://share.naver.com/web/shareView` 방식 기반으로 브라우저 라우팅 작업을 수행합니다.
- **Clova Face Recognition (CFR)**: 단순 텍스트 데이터가 아니라 원본 사진 바이너리나 이미지를 전송할 경우 인공지능이 이를 분해하여 해당 사진 안의 인물의 위치 픽셀, 성별 인식값, 표정에 따른 감정 비율 등을 측정해 수치로 되돌려줍니다. 연예인 얼굴 정보 비교 분석 시에도 매우 유효합니다. `vision/face` 또는 `vision/celebrity` 형태로 호출하게 됩니다.

정리하자면 시스템 구축 시 이러한 인증의 차이에서 오는 헤더 작성법을 주의하면서, 내가 설계하려는 피쳐(Feature)가 어떤 서비스 분류에 어울리는지를 면밀하게 판단하여 데이터랩, 검색 등 각 API 모듈의 URL을 매치해야 성공적인 설계가 완료됩니다.

# API 공통 가이드 (상세 원문본)

네이버 오픈API는 네이버 플랫폼 내부에 구축되어 있는 다양한 서비스 기술 요소와 핵심 기능을 외부 개발자가 자신의 서비스, 웹 페이지, 애플리케이션에 손쉽게 연동하고 통합적으로 사용할 수 있게 있도록 웹 상에서 RESTful 형태나 SDK 형태로 지원하고 공개하는 기술의 총칭입니다.

네이버 오픈API를 활용하여 사용할 수 있는 기술에는 여러 가지가 있습니다. 구체적으로는 네이버 사용자 계정을 연동시킬 수 있는 네이버 로그인, 지도 위치 데이터를 활용하는 네이버 지도, 실시간 방대한 데이터를 조회할 수 있는 검색 엔진 연동, 나아가 인공지능 기반의 기능을 지원하는 영역도 포함됩니다. 이 AI 기능들로는 Clova(클로바) 플랫폼을 기반으로 제공되는 뛰어난 음성 인식(Speech-to-Text) 및 음성 합성(Text-to-Speech) 기술, 얼굴 안면 정보의 다양한 특징을 인식하는 얼굴 인식 기술, 그리고 Papago(파파고)의 고성능 인공신경망 기계 번역 기술(NMT) 등이 대표적으로 오픈 API를 통해 배포되고 있습니다. 외부 개발자는 이러한 자원을 무상 혹은 약정된 규칙 하에서 활용하여, 풍부하고 품질 높은 서비스 생태계를 만들어갈 수 있도록 장려됩니다.

## API 공통 가이드 개요
API 공통 가이드는 네이버 오픈API를 처음 도입하고자 하는 개발자나 기존 환경을 유지 보수하려는 관리자 모두 네이버가 제공하는 클라이언트 애플리케이션 툴 키트를 다루기 위해 필수적으로, 또 미리 알아 두어야 하는 핵심 내용과 규칙을 자세히 설명하고 있습니다.
- 최종 수정일: 2021년 8월 27일
(이 안내서 등 관련 문서의 내용은 서비스 개선 및 갱신에 따라 향후 언제든지 사전 안내 없이 변경될 위험과 가능성이 있으므로 항상 최신 버전을 참고하시기 바랍니다)

## API 공통 가이드 구성 체계
본 가이드는 개발 진행의 허들을 낮추기 위해 여러 단계로 쪼개어 계층화된 안내를 합니다. 내용은 다음과 같은 주제들로 분리 구성되어 있습니다.

### 1. 네이버 오픈API 종류
네이버가 제공하고 있는 수많은 API를 기술 구현과 인증 방식을 뼈대로 양분한 내용을 담고 있습니다.
여기서는 사용자로부터 명시적으로 데이터 통제권을 위임받을 필요가 있는 '로그인 방식 오픈 API' 시스템들의 속성과, 단순 클라이언트 식별 정보만으로 네이버가 가진 정보 자체를 자유롭게 호출 가능한 '비로그인 방식 오픈 API' 두 가지 큰 줄기로 나눠서 기술합니다.

### 2. 사전 준비 사항
아무런 준비 상태 없이 무작정 API를 테스트 하거나 실행할 수 없으므로, 네이버 개발자 센터에서 진행해야만 하는 필수적인 첫 준비 과정을 안내합니다.
- 애플리케이션 등록: 프로젝트를 네이버 서버 내비게이션에 신규 안착시키는 절차.
- 애플리케이션 등록 세부 정보: 서비스의 형태(웹, iOS, Android 등)와 연동 콜백 주소 등을 알맞게 조정 등록하는 방법을 배웁니다.
- 애플리케이션 등록 확인: 등록이 원활하게 진행되었는지 스토어 또는 리스트업 상의 점검입니다.
- 클라이언트 아이디와 클라이언트 시크릿 확인: 네이버 인증 서버 혹은 API 게이트웨이와 통신할 때 고유하게 사용하는 여권/서명 키 세트를 발급받고 저장하는 과정입니다.

### 3. 내 애플리케이션 관리
애플리케이션이 구동 중이거나 개발 테스트가 진행 중일 때 다양한 현황과 부가적인 정보를 수정 및 확인하기 위한 관리 메뉴 설명 영역입니다.
- 기본 정보: 앱 이름이나 핵심 스펙 설명 및 가이드.
- API 설정: 최초 생성 시 빠트린 권한이나 새롭게 추가하려는 API 호출 권한의 세부 변수 제어.
- 멤버 관리: 권한이 격리된 개발 중 단계에서 API를 함께 테스트해 볼 개발 참여자 아이디 등의 테스터 권한 등록.
- 통계 보기: API가 어느 시점에 얼마나 많이 불리고 있는지 실시간 콜 수 데이터를 확인할 수 있는 기능 안내.
- 애플리케이션 삭제: 프로젝트 중단 또는 정리 시 필요한 탈퇴 스크립트 안내.

### 4. 용어 정리
초보자나 다른 개발 생태계에서 넘어온 작업자가 모호하게 느낄 수 있는 단어 혼용으로 인한 실수를 방지하는 부분입니다.
- API의 기본: Application Programming Interface의 가장 기본적 정의부터 재정립.
- API 인증: 왜 클라이언트 정보나 토큰이 필요한지 설명.
- API 호출: HTTP를 전제로 한 호스트 URL, URI 등의 규격 설명.
- HTTP: 호출 매개체와 통신 포맷 자체에 대한 지식 전달.

### 5. 예시 및 지원 자료
- 샘플 코드: 실제 서비스 단의 적용 전 테스트 단계에서 활용 가능한 로그인 방식 오픈 API 데이터의 호출 예 모음과 비로그인 오픈 API 방식의 언어별 호출 예를 제공하여 그대로 복사하기만 해도 되는 수준으로 제공.
- 오류 코드: 잘못된 매개변수나 서버 접속 이상으로 응답 시 HTTP 에러 외에 주요하게 쏟아져 나오는 전용 에러 코드의 종류 및 원인(주요 오류 코드 정리, 오류 메시지 응답 형식 등)에 대해 집중 조명합니다.

이 문서를 비롯하여 각각의 세부 문서가 수십 줄에 달하는 분량으로 독립적으로 묶여 있으니, 전체 구축 생태계 진입에 앞서 반드시 참고하시기 바랍니다.
